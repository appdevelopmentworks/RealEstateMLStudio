## 技術的負債・改善項目

### 優先度: 高

| 項目 | 詳細 | 対応策 |
|------|------|--------|
| テストコード不足 | ユニットテストなし | pytest導入 |
| エラーハンドリング | 一部try-except不足 | 網羅的に追加 |
| 型ヒント不足 | 一部関数のみ | 全関数に追加 |

### 優先度: 中

| 項目 | 詳細 | 対応策 |
|------|------|--------|
| ログ出力 | print文のみ | logging導入 |
| 設定ファイル | ハードコード | config.yaml化 |
| Docker対応 | 未対応 | Dockerfile作成 |

### 優先度: 低

| 項目 | 詳細 | 対応策 |
|------|------|--------|
| パフォーマンス最適化 | 大規模データ未対応 | キャッシュ導入 |
| コードドキュメント | docstring不完全 | 全関数に追加 |
| CI/CD | 未設定 | GitHub Actions |

## 貢献ガイドライン

### 新機能追加の手順

1. Issueを作成して機能を提案
2. 設計をドキュメント化
3. ブランチを作成（`feature/機能名`）
4. 実装・テスト
5. PRを作成
6. コードレビュー
7. マージ・リリース

### コーディング規約

- Python: PEP 8準拠
- インデント: 4スペース
- 最大行長: 100文字
- docstring: Google style
- 命名規則:
  - クラス: PascalCase
  - 関数/変数: snake_case
  - 定数: UPPER_CASE
